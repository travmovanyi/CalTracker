# Telegram-бот для підрахунку калорій та схуднення

## Що вміє бот
- Реєстрація профілю (стать, вік, зріст, вага, ціль, активність)
- Розрахунок денної норми калорій (формула Міффліна-Сан Жеора)
- Додавання прийомів їжі: пошук у вбудованій базі ~45 продуктів або ручне введення калорій
- Денний підсумок з'їденого та залишок до норми
- Трекер ваги з графіком динаміки (PNG-зображення)

## Встановлення

1. Встанови Python 3.10+ і залежності:
   ```bash
   pip install -r requirements.txt
   ```

2. Отримай токен бота:
   - Напиши в Telegram боту **@BotFather**
   - Команда `/newbot`, дай назву — отримаєш токен виду `123456:ABC-DEF...`

3. Встав токен одним із способів:
   - Відкрий `bot/config.py` і встав токен у `BOT_TOKEN`, **або**
   - Задай змінну середовища:
     ```bash
     export BOT_TOKEN="твій_токен_тут"
     ```

4. Запусти бота:
   ```bash
   python main.py
   ```

Бот створить файл бази даних `calorie_bot.db` автоматично при першому запуску.

## Структура проєкту
```
calorie_bot/
├── main.py                  # точка входу
├── requirements.txt
├── bot/
│   ├── config.py             # токен, шлях до БД
│   ├── database.py           # робота з SQLite
│   ├── calculations.py       # формули BMR/TDEE/норма калорій
│   ├── food_database.py      # локальна база продуктів
│   ├── keyboards.py          # клавіатури Telegram
│   ├── states.py             # FSM-стани сценаріїв
│   └── handlers/
│       ├── start.py          # /start, реєстрація, профіль
│       ├── food.py           # додавання їжі, денний підсумок
│       └── weight.py         # внесення ваги, графік
```

## Розгортання на Railway (безкоштовний хостинг)

1. **Створи репозиторій на GitHub** і завантаж туди весь вміст папки `calorie_bot` (файли `main.py`, `bot/`, `requirements.txt`, `Procfile` тощо).
   - Якщо GitHub ще немає: зареєструйся на [github.com](https://github.com), створи новий репозиторій (New repository), завантаж файли через кнопку "Add file → Upload files" прямо в браузері — без командного рядка.

2. **Зареєструйся на [railway.app](https://railway.app)** (можна через GitHub-акаунт).

3. Натисни **New Project → Deploy from GitHub repo** і обери свій репозиторій з ботом.

4. Railway автоматично визначить, що це Python-проєкт, встановить залежності з `requirements.txt` і запустить команду з `Procfile`.

5. **Додай токен бота як змінну середовища:**
   - У проєкті на Railway відкрий вкладку **Variables**
   - Додай змінну: `BOT_TOKEN` = `твій_токен_від_BotFather`
   - Railway автоматично передасть її боту (код уже читає `os.getenv("BOT_TOKEN")`)

6. Перейди у вкладку **Deployments → Logs** — маєш побачити повідомлення `Бот запущено`. Це означає, що бот працює і відповідає в Telegram навіть коли твій комп'ютер вимкнений.

⚠️ **Важливо про базу даних:** файл `calorie_bot.db` зберігається у файловій системі контейнера Railway, яка **не є постійною** — при кожному новому деплої (наприклад, після зміни коду) дані можуть зникнути. Щоб дані зберігались назавжди:
- У проєкті Railway додай **Volume** (Settings → Volumes → New Volume), примонтуй його, наприклад, до `/data`
- Зміни в `bot/config.py` шлях: `DB_PATH = "/data/calorie_bot.db"` (або зроби його теж змінною середовища)

Без цього кроку бот працюватиме, але при кожному оновленні коду історія їжі/ваги користувачів обнулиться.

## Як розширити
- **База продуктів**: додай нові записи у `bot/food_database.py` (формат: калорії, білки, жири, вуглеводи на 100 г).
- **Реальна база продуктів через API**: замість `food_database.py` можна підключити зовнішній сервіс (наприклад, Edamam або Open Food Facts) — знадобиться API-ключ.
- **Нагадування**: можна додати щоденне нагадування внести вагу через `aiogram` + `APScheduler`.
- **Хмарний хостинг**: бот можна розгорнути на будь-якому сервері (VPS, Railway, Render) — просто тримай процес `python main.py` запущеним постійно.

## Важливо
Розрахунок норми калорій — орієнтовний і базується на стандартних формулах. Мінімальні безпечні пороги (1500 ккал для чоловіків, 1200 для жінок) вбудовані, щоб уникнути надто різкого дефіциту. Для індивідуальних медичних рекомендацій варто звернутись до лікаря або дієтолога.
